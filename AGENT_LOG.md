# AGENT_LOG

An honest account of how this project was built with an AI coding agent.

## Which agent

- **Claude Code** (Anthropic's CLI agent, Claude Opus) drove the implementation
  end-to-end, with human direction on architecture, scope, and review.

## What the agent scaffolded

Essentially the full source tree was generated by the agent, in dependency order
so each layer compiled against the last:

1. **Build config** ‚Äî installed all dependencies; wired `babel-plugin-module-resolver`
   (`@/` alias) + the Reanimated babel plugin; added `react-native-gesture-handler`
   to the entry; enabled `strict` + path aliases in `tsconfig`.
2. **Design system** ‚Äî `constants/` (colors + gradients, typography on SF Pro,
   spacing/radius/motion, centralised strings).
3. **Types** ‚Äî `Task` / `Subtask` and all four agents' typed result shapes.
4. **Storage** ‚Äî a typed AsyncStorage wrapper + `taskStorage` / `settingsStorage`.
5. **AI service** ‚Äî REST `anthropicClient` (retry/timeout/typed errors/structured
   output) + the four agent modules, each with its own system prompt.
6. **State** ‚Äî Zustand `taskStore` / `settingsStore` with write-through persistence.
7. **Hooks** ‚Äî `useTasks` + one hook per agent managing loading/clarify/error/success.
8. **Components** ‚Äî ~20 reusable animated primitives (GlassCard, GradientButton,
   GlowRing, TypingIndicator, TypewriterText, Shimmer, AnimatedCheckbox, FilterTabs,
   SelectablePills, ErrorBoundary, ‚Ä¶) and feature components (TaskCard with
   swipe-to-delete, FAB, AI result views).
9. **Screens** ‚Äî list, detail, create/edit form, AI tab, settings.
10. **Navigation + providers** ‚Äî native stack + bottom tabs, dark theme, bootstrap
    hydration gate, splash, root error boundary.
11. **Docs + tests** ‚Äî README, this log, `.env.example`, jest unit tests.

After generation: `tsc --noEmit`, `eslint`, and `jest` all run clean.

## What needed human judgment / manual adjustment

- **Model choice.** The assignment pins `claude-sonnet-4-6` over REST, so the agent
  used axios directly rather than the SDK or the newer default model ‚Äî a case where
  the explicit spec overrides the agent's default instinct.
- **Structured output strategy.** Chose the Messages API `output_config.format`
  (JSON schema) for reliable typed agent results, but still wrapped parsing in a
  defensive extractor (`parseJSON`) that survives code fences / prose ‚Äî belt and
  braces, and unit-tested.
- **Multi-step agent design.** Deciding *which* agents genuinely need a clarifying
  round (Prioritization, Decomposition) vs. a single classify-then-write pass
  (Status Update, Blocker Detector) was a product call, not something to leave to
  the model. The clarify hand-off (persisting the conversation in a ref and feeding
  the answer back) was designed deliberately.
- **Navigation trade-off.** The spec asked for "native stack with custom spring
  interpolators". `@react-navigation/native-stack` uses platform-native transitions
  and does not expose `cardStyleInterpolator` (that's the JS stack). Rather than pull
  in the JS stack, the native stack's animation presets were used and the spring/
  fade-scale character was layered on with in-screen Moti entrances ‚Äî a conscious
  trade-off favouring native performance, noted here for honesty.
- **Android blur.** `@react-native-community/blur` is uneven on Android, so glass
  surfaces fall back to a translucent surface there instead of a hard blur.
- **API key storage.** Kept device-local in AsyncStorage (per the single-user,
  local-storage intent) rather than the secure keychain; flagged as a production
  hardening step. `.env.example` documents the variable for teams that prefer
  build-time injection via react-native-config.
- **Type friction.** `react-native-linear-gradient` rejects `readonly` color tuples,
  so the gradient constants are typed as mutable `string[]` ‚Äî a small but real
  adjustment caught by the strict typecheck.
- **Testing scope.** The default full-app render test was replaced with focused unit
  tests on pure logic (sort/filter/progress + the AI JSON parser). Rendering the full
  tree in jest would require mocking a stack of native modules (reanimated, gesture-
  handler, blur, svg) for little signal; testing the business logic is the higher-
  value, reliable choice here.

## Where AI helped most vs. least

- **Most:** breadth and consistency ‚Äî generating a large, internally-consistent,
  strictly-typed codebase with a uniform design system and matching animation idioms
  far faster than by hand, and keeping imports/barrels/aliases coherent across ~60 files.
- **Least / human-led:** product decisions (agent step design, which feature is the
  "custom" one and why it's useful), API-contract choices, and the honest trade-offs
  above. The agent is excellent at execution and consistency; the architectural and
  product framing is where human judgment carried the work.
